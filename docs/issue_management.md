# 이슈 관리 가이드

이 프로젝트의 모든 이슈는 `docs/issues/NNN-title.md` 형식으로 관리하며, 아래의 우선순위와 상태 체계를 따릅니다.

## 우선순위 (Priority)

| 등급 | 설명 | 대상 예시 |
| :--- | :--- | :--- |
| **Critical** | 시스템 사용 불가 또는 데이터 손실 위험 | 서버 다운, 결제 오류, 심각한 보안 취약점 |
| **High** | 주요 기능의 결함 또는 성능 저하 | 이미지 업로드 실패, 현저하게 느린 반응 속도 |
| **Normal** | 일반적인 버그 또는 개선 사항 | 레이아웃 미세 조정, 부가 기능 추가 |
| **Low** | 사소한 개선 또는 장기적 검토 사항 | 오타 수정, 비본질적인 브랜딩 개선 |

## 상태 (Status)

- `open`: 이슈 발생 및 검토 중
- `in-progress`: 작업 진행 중
- `resolved`: 해결 완료 (병합 대기)
- `closed`: 해결 후 검증 완료 또는 반려

## 사고 대응 및 복구 절차 (Incident Recovery)

### 1. 마이그레이션 실패 및 레이아웃 회귀
- **증상**: 리팩토링 후 기존 레이아웃이나 라벨이 훼손된 경우.
- **대응**: 즉시 작업을 중단하고 `git checkout -f main`으로 원복합니다. '원본 코드 심층 분석' 단계부터 다시 수행하며 픽셀 단위 대조 절차를 밟습니다.

### 2. 환경 변수(.env) 및 로컬 파일 유실
- **증상**: `git clean` 등의 명령어로 환경 설정이 사라진 경우.
- **대응**: 
    1. Vercel 대시보드의 `Environment Variables`에서 최신 키 값을 복구합니다.
    2. 로컬 이미지 폴더 유실 시 휴지통 또는 외부 백업을 확인합니다.
    3. 복구 후 반드시 `node api/server.js`로 통합 서버를 재가동하여 서비스 정상 여부를 검증합니다.

### 4. 분석 데이터 누락 및 추적 실패
- **증상**: GA 대시보드에 특정 이벤트(예: 공유, 게시) 수치가 비정상적으로 낮거나 수집되지 않는 경우.
- **대응**:
    1. 브라우저 콘솔을 열고 `[Analytics]` 로그가 출력되는지 확인하여 `Analytics Manager` 가동 여부를 점검합니다.
    2. `template.html`에 삽입된 GA 측정 ID(`G-48JCX4VZXD`)를 확인합니다.
    3. 로컬 빌드 후 `dist/template.html`에 스크립트가 유지되고 있는지 검증합니다.
    4. 네트워크 탭에서 `google-analytics.com/collect` 요청의 상태 코드가 `200`인지 확인합니다.
