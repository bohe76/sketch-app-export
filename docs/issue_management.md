# 이슈 관리 가이드

이 프로젝트의 모든 이슈는 `docs/issues/NNN-title.md` 형식으로 관리하며, 아래의 우선순위와 상태 체계를 따릅니다.

## 우선순위 (Priority)

| 등급 | 설명 | 대상 예시 |
| :--- | :--- | :--- |
| **Critical** | 시스템 사용 불가 또는 데이터 손실 위험 | 서버 다운, 결제 오류, 심각한 보안 취약점 |
| **High** | 주요 기능의 결함 또는 성능 저하 | 이미지 업로드 실패, 현저하게 느린 반응 속도 |
| **Normal** | 일반적인 버그 또는 개선 사항 | 레이아웃 미세 조정, 부가 기능 추가 |
| **Low** | 사소한 개선 또는 장기적 검토 사항 | 오타 수정, 비본질적인 브랜딩 개선 |

## 상태 (Status)

- `open`: 이슈 발생 및 검토 중
- `in-progress`: 작업 진행 중
- `resolved`: 해결 완료 (병합 대기)
- `closed`: 해결 후 검증 완료 또는 반려
