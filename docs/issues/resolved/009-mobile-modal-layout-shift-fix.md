## 이슈 009: 모바일 상세 모달의 레이아웃 점프(Layout Shift) 현상 해결

## 현상

- 공유 링크 또는 모바일 피드 진입 시, 상세 모달이 열릴 때 이미지 로딩 전까지는 높이가 0에 가까웠다가 로딩 직후 '퍽' 하고 커지는 시각적 부조화(Layout Shift) 발생.
- 이는 사용자에게 불안정한 경험을 주고 웹 바이탈(CLS) 점수에도 악영향을 줌.

## 원인 분석

- 모달 컨테이너(`h-auto`)가 내부 콘텐츠(이미지)의 높이에 전적으로 의존하는 구조임.
- 이미지가 없는 초기 상태에서는 헤더와 푸터만 렌더링되어 높이가 매우 낮게 설정됨.

## 해결 방안

1. **고정 높이 전략 (Fixed Height Policy)**:
   - 모바일 환경에서 모달 컨테이너의 높이를 `h-[85vh]`로 고정하여, 이미지 로딩 여부와 관계없이 절대적인 레이아웃 안정성을 확보함.
   - 이는 화면 상하단에 약 7.5%씩의 균형 잡힌 여백(Backdrop)을 제공하여 시각적 완성도를 높임.

## 결과 및 효과

- 모달이 열릴 때부터 닫힐 때까지 크기 변화가 0 (Zero Layout Shift)으로 유지됨.
- 이미지는 1.2초 동안 부드럽게 페이드인(Fade-in) 되며, 모달 창의 흔들림이 전혀 없어 사용자에게 견고하고 고급스러운 느낌을 줌.
