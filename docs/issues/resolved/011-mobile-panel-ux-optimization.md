## 이슈 011: 모바일 스타일 패널 UX 전면 개편 (가림 현상 해결 & 틴트 피커 고도화)

## 현상

1.  **화면 가림 (Occlusion)**: 모바일의 좁은 화면에서 세로형 스타일 리스트가 캔버스(그림)를 가려, 사용자가 스타일 변경 결과를 즉시 확인하기 어려움.
2.  **반응성 저하 (Latency)**: UI 애니메이션을 위한 인위적인 딜레이(0.5s Pause)로 인해, 스타일 변경 시 답답함을 유발함.
3.  **UI 결함 (Artifacts)**: `react-colorful` 틴트 피커 하단에 미세한 1px 라인이 생기거나, 터치 드래그 시 높이가 흔들리는 현상 발생.

## 해결 방안

1.  **가로 썸네일 뷰 (Horizontal Scroll View)**:
    - 하단에 3개의 스타일 썸네일을 가로로 배치하여 상단 캔버스 영역을 최대한 확보함.
    - 선택된 스타일 정보는 상단 헤더에 동적으로 표시.
2.  **즉시 반응 (Zero Latency)**:
    - 모바일 전용 딜레이 로직을 제거하고, 스타일 선택 즉시 그리기 엔진을 업데이트하여 빠릿한 체감을 제공.
    - `Classic`/`Vivid` 선택 시 패널을 즉시 닫고, `Vintage` 선택 시에만 틴트 피커를 유지함.
3.  **오버레이 틴트 피커 (Overlay Tint Picker)**:
    - `Vintage` 모드에서만 스타일 패널 위로 독립적인 틴트 피커가 오버레이됨.
    - CSS Injection(`margin-top: -1px`, `height: 120px !important`)을 통해 1px 틈새와 레이아웃 흔들림(Shift)을 원천 차단함.

## 결과 및 효과

- **편집 경험 개선**: 사용자가 자신의 그림을 가리지 않고 보면서 실시간으로 스타일을 변경할 수 있게 됨.
- **완성도 향상**: 픽셀 단위의 UI 마감(Pixel-Perfect)으로 앱의 퀄리티를 한 단계 높임.
